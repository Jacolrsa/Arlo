# Development Rules

These rules apply to every version of Arlo.

1. Always generate COMPLETE files.

2. Never generate partial edits unless explicitly requested.

3. Never tell the developer to edit code manually.

4. Every file must compile before moving to the next step.

5. Architecture changes only occur during framework versions.

6. Feature versions do not change the architecture.

7. Every release must be committed to Git.

8. Every release receives a version tag.

9. Documentation is updated with every completed version.

10. Home Assistant best practices must always be followed.
